## Conceitos Fundamentais

Gestão de Conformidade de Segurança envolve garantir que a organização atenda a requisitos legais, regulatórios e contratuais de segurança. Inclui gestão de auditorias, evidências e programas de conformidade contínua.

## Frameworks de Referência

- **ISO 27001:2022**: 93 controles em 4 temas (Organizacionais, Pessoas, Físicos, Tecnológicos)
- **LGPD (Lei 13.709/2018)**: Proteção de dados pessoais no Brasil — DPO, base legal, DPIA
- **PCI DSS v4.0**: 12 requisitos para proteção de dados de cartão

## Controles Principais

1. Programa de Conformidade Contínua
2. Gestão de Auditorias Internas e Externas
3. Registro de Tratamento de Dados (LGPD Art. 37)
4. Análise de Lacunas (Gap Analysis)
5. Gestão de Evidências de Controles

## KPIs do Domínio

| Métrica | Fórmula | Benchmark |
|---------|---------|-----------|
| Taxa de conformidade ISO | Controles implementados / Total controles | ≥ 80% (certificação) |
| Não-conformidades abertas | NC abertas há > 90 dias | 0 NC críticas |
| Cobertura LGPD | Processos mapeados / Total processos com dados pessoais | 100% |

## Aplicação por Situação

- **S14 (ISO 27001)**: Gap analysis → plano de tratamento → implantação controles → auditoria interna → certificação
- **S17 (LGPD/GDPR)**: Mapeamento de dados → base legal → DPIA → DPO → plano de adequação
